create-next-quick

A lightning-fast CLI tool to scaffold Next.js projects with your choice of options

Get Started View on GitHub

Lightning Fast

Instantly scaffold Next.js projects with minimal configuration and maximum speed.

Flexible Options

Choose TypeScript, Tailwind CSS, App Directory, linters, ORMs, and more through interactive prompts.

Smart Defaults

Sensible defaults for modern Next.js development with the ability to customize everything.

Package Manager Detection

Automatically detects installed package managers (npm, yarn, pnpm) and prompts accordingly.

Multiple Pages

Create multiple pages at once to jumpstart your project structure.

Safe Creation

Prevents accidental overwrites by checking directory contents before creation.

Why create-next-quick?
create-next-quick is a lightweight and fast alternative to create-next-app with additional features like multiple page generation, ORM setup, and enhanced customization options.

Installation

You don't need to install create-next-quick globally. Run it instantly with npx:

bash 
npx create-next-quick

Requirements

  • Node.js 16.x or higher
  • npm, yarn, or pnpm
Tip: The tool will automatically detect which package managers you have installed and only show available options.

Quick Start

Basic Usage

bash 
# Create a new project with interactive prompts
npx create-next-quick

# Create with project name (skips name prompt)
npx create-next-quick my-awesome-app

Interactive Setup

When you run the command, you'll be prompted to configure your project:

interactive 
? Enter project name: my-portfolio
? Choose a package manager: pnpm
? Do you want to use TypeScript? Yes
? Do you want to use Tailwind CSS? Yes
? Do you want to use src directory? Yes
? Do you want to use the app directory? Yes
? Enter the names of the pages you want to create: home, about, contact
? Choose a linter: eslint
? Choose an ORM: prisma
? Do you want to use Shadcn UI? Yes

What Gets Created

  • A clean Next.js project with your selected options
  • Custom pages if specified
  • Configured linting tools
  • ORM setup with boilerplate code
  • Shadcn UI integration (if selected)
  • Clean public folder (removes default favicon)
  • Empty default page template
Next Steps:
After creation, navigate to your project directory and run npm run dev to start the development server.

Interactive Prompts

Option Description Default Choices

Tailwind CSS Include Tailwind CSS Yes Yes/No
Src Directory Use src/ directory structure Yes Yes/No
App Directory Use Next.js 13+ app directory Yes Yes/No

API Reference

Core Functions

run(command, cwd, silent)

Executes shell commands with proper error handling.

  • command - Command to execute
  • cwd - Working directory (optional)
  • silent - Suppress output (optional)

createPages(pagesPath, pages, useTypeScript, useAppDir, useSrcDir)

Generates custom pages based on user input.

createLayout(projectPath, projectName, useTypeScript, useAppDir, useSrcDir)

Creates the root layout file with project metadata.

Validation Functions

validateProjectName(input)

Validates project name according to rules:

  • Must be lowercase
  • Cannot overwrite existing directories
  • Current directory must be empty when using "."

Frequently Asked Questions

General Questions

Q: How is this different from create-next-app?

A: create-next-quick offers additional features like multiple page creation, ORM setup, enhanced customization options, and package manager detection while maintaining the simplicity of create-next-app.

Q: Can I use this in production projects?

A: Absolutely! create-next-quick generates standard Next.js projects with best practices and is suitable for production use.

Q: What Node.js version is required?

A: Node.js 16.x or higher is required, which aligns with Next.js requirements.

Usage Questions

Q: Can I create a project in the current directory?

A: Yes, use npx create-next-quick . but ensure the current directory is empty.

Q: How do I add more pages after creation?

A: You can manually create pages following the same structure as generated pages, or run the tool again in a different directory.

Q: Can I skip certain prompts?

A: Currently, all prompts are interactive, but you can quickly accept defaults by pressing Enter.

Technical Questions

Q: Which package managers are supported?

A: npm, yarn, and pnpm are all supported. The tool automatically detects which ones you have installed.

Q: Can I use both Prisma and Drizzle?

A: No, you can only choose one ORM per project. However, you can manually add another ORM later if needed.

Q: Does it work with the latest Next.js version?

A: Yes, create-next-quick uses the latest version of create-next-app under the hood, ensuring compatibility with the newest Next.js features.

Troubleshooting

Q: I'm getting a "directory not empty" error

A: This happens when trying to create in a non-empty directory with ".". Either choose a different name or clean the current directory.

Q: The tool seems to hang during installation

A: This usually happens during package installation. Check your internet connection and package manager configuration.

Q: Some features aren't working as expected

A: Make sure you're using the latest version by running the command again. If issues persist, check the GitHub repository for known issues.

Need Help? If you can't find an answer here, feel free to open an issue on the GitHub repository.
